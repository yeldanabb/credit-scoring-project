# Credit Scoring Model (PD Prediction) 

Проект по разработке системы кредитного скоринга для оценки вероятности дефолта (Probability of Default). Модель построена на данных LendingClub с использованием классических банковских методологий как WOE и IV и современных ML-инструментов.

## Основные результаты
- **Gini Index:** 0.394 (Logistic Regression) | 0.397 (XGBoost)
- **KS Statistic:** 0.28
- **Модель:** Logistic Regression (выбрана за интерпретируемость и стабильность).
- **Инфраструктура:** FastAPI сервис, упакованный в Docker.

## Этапы реализации

### 1. Подготовка данных и SQL (Lite-Engineering)
Для симуляции работы с банковским хранилищем данных была использована база данных SQLite. Был настроен ETL-процесс и выполнена фильтрация заемщиков, исключены активные кредиты по которым еще невозможно определить факт дефолта. Целевая выборка извлечена с помощью SQL запроса, который включает только завершенные кейсы или серьезные просрочки.

### 2. Target Engineering
Создана целевая переменная на основе статусов `loan_status`

### 3. WOE и IV Анализ
Для обеспечения линейности и интерпретируемости модели проведена трансформация признаков через **Weight of Evidence**. Рассчитан **Information Value** для отбора признаков.
Лучшие признаки: `int_rate`, `installment_to_inc`, `int_rate`, `term_int`, `loan_to_inc`, `dti`.

### 4. Моделирование и Валидация
Сравнение логистической регрессии на WOE-признаках и градиентного бустинга. Валидация проводилась по стандартам Gini и KS.

## Запуск сервиса

Сервис позволяет отправлять анкету клиента в формате JSON и мгновенно получать решение банка.

### Запуск через Docker:
```bash
docker-compose up --build
```

### Тестирование API:
После запуска перейдите на http://localhost:8000/docs .

Пример запроса:

```bash
{
  "loan_amnt": 5000,
  "term": "36 months",
  "int_rate": 11.5,
  "installment": 160,
  "annual_inc": 55000,
  "dti": 15,
  "revol_util": 30,
  "purpose": "debt_consolidation"
}
```

Пример ответа:

```bash
{
  "probability_of_default": 0.1238,
  "decision": "Approve",
  "score": 876
}
```

### Структура проекта
- notebooks/: Исследовательский анализ и обучение моделей.
- app/: Код FastAPI сервиса.
- models/: Сохраненные артефакты (модель, bins, список признаков).
- Dockerfile & docker-compose.yml: Настройки контейнеризации.
